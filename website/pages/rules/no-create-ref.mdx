# no-create-ref

## Rule category

Restriction.

## What it does

Prevents usage of `createRef()` in function components.

## Why is this bad?

`createRef()` is a legacy API that is not recommended for use in new code. Instead, prefer using `useRef()` with function components.

## Examples

### Failing

```tsx twoslash
import React, { createRef } from 'react';

interface ExampleProps {}

function Example({}: ExampleProps) {
// @error: Do not use 'createRef'
  const ref = React.createRef<HTMLDivElement>();

  return <div ref={ref} />;
}
```

### Passing

```tsx twoslash
import React, { useRef } from 'react';

interface ExampleProps {}

function Example({}: ExampleProps) {
// @annotate: Use 'useRef' instead
  const ref = useRef<HTMLDivElement>(null);

  return <div ref={ref} />;
}
```

```tsx twoslash
import React, { createRef } from 'react';

interface ExampleProps {}

class Example extends React.Component<ExampleProps> {
  inputRef = createRef();
  // ...
}
```
